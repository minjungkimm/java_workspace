Thread - 하나의 프로그램에서 동시에
	     여러작업을 진행해보자!!
	     하나의 인간이 마치 분신을 여러개
	     두는 것과 같다...
단점 : 엉킨다, 동기화가 불안해진다..
동기화 : 일치시키는 것

자바에서 말하는 쓰레드:
하나의 프로세스내에서 독립적으로 실행되는
세부실행단위...

개발자는 독립적으로 실행하고 픈 로직을
run에 넣어라!!!!

Thread의 생명주기(언제 태어나서? new 할때
			어느때 일하고? JVM에게 맡길때, start()
			언제 죽나? run()메서드가 종료되면... )
**stop(), join(); 도 있지만.. Thread 를 개발자가 직접 쓰레드를
컨트롤한다는건 위험해서.. 존재하지만 특별한 경우에만 쓰인다..

**Thread 를 죽이는 방법!
public void run(){
	while(변수){ //변수값을 false로 돌려놓는다..
	}
	int x=3; //실행부가 while문 만나면 못빠져나와서 만날수없다
}

Thread 클래스 정의 방법 3가지
1.extends Thread - 내가 쓰레드가 되는 방법 --> .java
2.implements Runnable - 상속받다 -->.java
3.내부익명 -->클래스내에서

.java 만들지 VS 클래스내에서 할지
는 재사용성으로 결정!!

- Runnable
메인메서드가 new 한 Thread 를 start 하여 맡겨지면
JVM 에게 맡겨지는 곳 
Thread 는 동시에 수행되야 하므로, 하나씩 일부만 수행되면서 전부 돌아간다
Running 은 지금 수행되고 있는 메서드!! 
run 메서드 전부수행되면 dead

- Non-Runnable
Sleep 은 자동 복귀기능이 있기 때문에 순간, Non-Runnable 에 왔다가
돌아간다
